# Monet Index — ядро разворота

Не «угадай цену ника», а **поисковик вариаций + кросс-маркет цена**:
вбил/вставил любой подарок → видишь, что это за комбинация (модель × фон × узор),
её редкость и **где она дешевле всего сразу по всем площадкам Telegram**.

Оценщик стал распознавателем. Агрегатор цен — кассой. Полнота каталога — моатом.

## Что уже работает без тебя

Открой `monet-index.html` — прототип живой:
- поиск по названию коллекции и по имени трейта («Aurora», «Golden»);
- распознавание ссылок (`t.me/nft/PlushPepe-123`, `fragment.com/...`);
- сборка точной вариации → расчёт редкости комбинации («≈ 1 из N»);
- честный fair-value (флор × капнутый множитель редкости, с диапазоном);
- панель «дешевле всего по площадкам» с кнопками «купить».

Сейчас каталог = seed, цены = demo (детерминированные, **честно помечены** плашкой).
Вся логика, UX и адаптерный слой — настоящие. Меняется только источник данных.

## Граница: где без тебя никак

Две вещи требуют твоей инфраструктуры (песочница до них не дотягивается):

### 1. Полный каталог — запусти ингест
```bash
# на машине с обычным интернетом
TONAPI_KEY=твой_ключ node ingest.mjs        # бесплатный ключ на tonapi.io
```
Перед запуском впиши реальные TON-адреса коллекций в `STARTER` внутри `ingest.mjs`
(или передай свой список: `node ingest.mjs --collections collections.json`).
Скрипт страницами обходит все NFT коллекции, считает частоты трейтов, пишет
`catalog.json` с `_meta.seed = false`. Положи рядом с html — приложение само подхватит
и переключит бейдж на «каталог live».

### 2. Живые цены — задеплой воркер
```bash
npm i -g wrangler
wrangler secret put TONNEL_AUTH      # authData Tonnel
wrangler secret put PORTALS_TOKEN    # если есть доступ
wrangler secret put GETGEMS_KEY
wrangler deploy
```
Потом впиши URL воркера в `monet-index.html`:
```js
const CONFIG = { PROXY_BASE: "https://твой-воркер.workers.dev", ... };
```
Бейдж цен станет «live», hero-цена начнёт показывать реальную «дешевле всего на X».

Адаптеры в `worker.js` — это реальные `fetch + normalize` функции с размеченным
слотом под авторизацию. Под каждую площадку доточи детали запроса/парсинга по её
текущему API; формат выхода `normalize()` не трогай — тогда приложение остаётся
независимым от площадок.

## Почему так разбито

- **Полнота каталога** достижима программно (метадата подарков стандартизирована на
  ончейне) — поэтому это скрипт, а не ручной ввод коллекций.
- **Кросс-маркет цена** — единственное, чего нет ни у одной площадки (каждая видит
  только свой стакан). Нейтральный воркер-агрегатор = и есть продукт и моат.
- Адаптерный слой означает: добавление новой площадки = одна функция, фронт не трогаем.

## Монетизация — affiliate реф-метки (уже встроено)

Каждая кнопка «купить» прогоняется через `refLink()` и тегается твоим реф-кодом —
зарабатываешь на сделках, которые привёл. Заполни в `monet-index.html`:
```js
REF: {
  Tonnel:  { param:"startapp", code:"твой_код" },
  Portals: { param:"ref",      code:"твой_код" },
  Getgems: { param:"ref",      code:"твой_код" },
  Fragment:{ param:"",         code:"" }   // у Fragment нет реф-программы
}
```
Клики по «купить» шлются (sendBeacon) на `?track` воркера — там копятся в KV для
атрибуции, сколько сделок ты реально привёл. Пустые коды → чистые ссылки, ничего не ломается.

## Снайпер-алерты (клиент готов, бот — твой)

В карточке вариации есть блок «③ Снайпер-алерт»: ставишь целевую цену → сохраняется
локально и (если есть прокси + Telegram) синкается в воркер `?watch`.
Серверная часть в `worker.js`:
- `?watch` — принимает подписки, валидирует личность через `verifyInitData` (HMAC по BOT_TOKEN);
- `scheduled()` — крон обходит подписки, проверяет минимальную цену по площадкам, пингует бота при попадании в цель.

Что добавляешь ты (та же граница инфраструктуры):
```bash
wrangler kv namespace create WATCH_KV       # привяжи как WATCH_KV в wrangler.toml
wrangler secret put BOT_TOKEN               # токен твоего Telegram-бота
# wrangler.toml:
#   [triggers]
#   crons = ["*/5 * * * *"]                 # проверять каждые 5 минут
wrangler deploy
```
